# Data Schema — 周报数据契约

生成一份周报需要的全部输入。三种用法（优先级从高到低）：

- **Markdown 源（主）**：用户给一份 Markdown 周报文档，AI 按 [`markdown-source.md`](markdown-source.md) 的结构约定/字段映射抽取——最常用，本契约的字段即 markdown 的映射目标。
- **JSON 填表式**：用户照 `templates/data.example.json` 填好，交给 AI 渲染。
- **AI 访谈式**：AI 按下面「访谈提纲」逐项问用户，边问边填。

原则：**缺的就标 `—`，不要编造**。一份诚实的报告里有空位，好过一份漂亮的报告里有假数。

---

## 访谈提纲（最少必答）

1. **你的团队/业务线叫什么？本周是哪个周期？数据口径有什么注意点？**
2. **如果只能说一句话，本周最重要的事是什么？**——逼出「但是」：好消息背后的张力 / 隐忧。
3. **本周的赢点是什么？最大的风险是什么？最值得加注的方向是什么？** 每个给一个能代表它的数字。
4. **挑 4–6 个你每周都盯的核心指标**，各自这周的值和环比。
5. **有没有一条值得画成趋势线的指标？**（近 8–12 期）
6. **收入/成本/用户能拆成几块占比吗？**（≤5 块）
7. **能按地区/产品线/渠道/客户分层拆一张明细表吗？**
8. **有哪些指标该报警或重点跟进？**（当前值、阈值、状态）
9. **下周要做的 2–4 件事，分别谁负责、什么时候交？**

## 字段说明

### `identity` — 身份与周期
| 字段 | 说明 | 示例 |
|---|---|---|
| `team` | 团队/公司名 | "经纬数字资产" |
| `subtitle` | 报告类型 | "CEO 周度经营复盘" |
| `period` | 报告周期 | "2026 第 23 周 · 6/8 – 6/14" |
| `caveat` | 数据口径 | "口径 T+1" |
| `accent` | 主色（换肤，可省，默认深青） | "#0f766e" |

### `headline` — 一句话结论
| 字段 | 说明 |
|---|---|
| `verdict` | 核心判断，含「但是」。可用 `<em>` 标正向、`<span class="warn">` 标风险 |
| `summary` | 一行关键数字摘要。⚠️ 当前值与阈值分清（「CAC 升至 $58、突破 $50 红线」） |

### `pillars` — 三支柱（数组，固定 3 个：win / risk / bet）
| 字段 | 说明 | 示例 |
|---|---|---|
| `kind` | `win`｜`risk`｜`bet` | "win" |
| `tagText` | 标签文字 | "本周赢点" |
| `title` | 支柱标题 | "交易量创新高" |
| `value` / `unit` | hero 数字 + 单位 | "48.2" / "亿$" |
| `delta` | 一行变化描述 | "12.3% 环比 · 8 周新高" |
| `deltaDir` | `up`｜`down`｜`cost-good`｜`cost-bad`｜`muted`（按 design-spec §4 判定树选；成本上升用 `cost-bad`，常规波动用 `muted`） | "cost-bad" |
| `spark` | 8 个相对高度(%) | [48,58,62,54,68,80,74,100] |
| `note` | 一段说明 | "BTC 行情 + 机构入金双驱动…" |

### `kpis` — 计分卡（数组，4–6 个）
| 字段 | 说明 | 示例 |
|---|---|---|
| `label` | 指标名 | "周交易量" |
| `value` / `unit` | 值 + 单位 | "48.2" / "亿$" |
| `wow` | 环比文字 | "12.3% 环比" |
| `wowDir` | `up`｜`down`｜`cost-good`｜`cost-bad`｜`muted`（同 `deltaDir`，见 design-spec §4 判定树） | "up" |

### `trend` — 趋势折线
| 字段 | 说明 | 示例 |
|---|---|---|
| `title` / `hint` | 卡片标题 / 口径副标题 | "周交易量趋势（近 8 周）" / "单位：亿美元" |
| `labels` | X 轴标签 | ["W16",…,"W23"] |
| `values` | 数值序列（AI 据此重算坐标） | [36.1,39.4,…,48.2] |

### `composition` — 结构环图
| 字段 | 说明 |
|---|---|
| `title` / `hint` | 标题 / 口径副标题（写口径，不写结论） |
| `centerValue` / `centerUnit` | 环心总数 + 单位 |
| `segments` | 数组：`{name, pct, color}`，≤5 段，色用主色明度阶；AI 据 pct 重算 dasharray/offset |

### `breakdown` — 明细表
| 字段 | 说明 |
|---|---|
| `title` / `hint` | 标题 / 口径 |
| `columns` | 表头数组 |
| `rows` | 行数组；含数值、环比（`up/down`）、状态 pill（`g/a/r` + 文字） |

### `risks` — 风险表
| 字段 | 说明 |
|---|---|
| `title` / `hint` | 标题 / 说明 |
| `rows` | 数组：`{metric, current, threshold, status:{level:g|a|r, text}}` |

### `nextSteps` — 下周行动（数组，2–4 个）
| 字段 | 说明 | 示例 |
|---|---|---|
| `urgency` | 时效徽章文字 | "即刻" / "本月" |
| `urgent` | 是否紧急（红徽章） | true |
| `text` | 行动描述，动词开头，关键词加粗 | "复盘获客渠道 ROI，砍掉单渠道 CAC>$80 …" |
| `owner` | 责任人 · 截止 | "增长负责人 · 6/18 前" |

### `footer`
口径 / 免责小字，如 "本页为虚构演示数据 · 仅用于版式评审"。

---

## 校验清单（AI 渲染前自检）

- 分区/分段表的数值之和、占比之和是否自洽（占比合计 100%、分项合计 = 总量）。
- 环图各段 pct 之和 = 100%，重算后 dash 之和 ≈ 439.8。
- 头部、支柱、KPI、表格里同一指标的数字与阈值是否完全一致。
- 每个 `deltaDir/wowDir` 是否按 design-spec §4 判定树选对：被判告警且方向一致用 `up/down`；成本类方向冲突用 `cost-good/cost-bad`；常规波动/无历史用 `muted`。
