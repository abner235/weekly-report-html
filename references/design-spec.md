# Design Spec — 一页纸周报渲染契约

> 这是让**每个团队的周报长得一致**的设计契约。生成任何报告前先读它。
> 原则优先于规则：每条约定都附了「为什么」，理解动机后可在同一设计语言内灵活处置；
> 但带 **🔒** 的项是硬约束（一致性 / 可访问性 / 正确性所系），不要随意发挥。

## 1. 设计哲学

**给的是判断，不是数据搬运。** CEO/业务负责人看周报的时间以秒计。报告的价值在于：第一眼看到**结论**，往下能找到**证据**，最后知道**下一步做什么**。所以全篇按 **结论先行 → 证据在后 → 行动收尾** 组织。

三个词概括风格：

- **克制（Restraint）** — 一个主色、留白充分、不堆装饰。颜色和加粗是稀缺资源，只用在需要被一眼抓住的地方。
  - 🔒 **颜色是稀缺的告警信号，不是涨跌的自动着色**。当一页里出现大量同向变化（如多条业务线在高基数后正常回调），不要把它们全部标红/标绿——那会让真正需要 CEO 警觉的那一项被淹没。判定规则：只有**头部结论判定为「需跟进 / 异常」的项**才用语义色（红/主色），被结论归类为「正常波动 / 非趋势性」的回调一律用**中性灰**（去箭头、保留 `+/-` 号）。让红色在整页里只出现在该被看见的地方。正向增长可保留主色，因为正面信号本身稀缺。
- **清晰（Clarity）** — 每个数字对齐、带单位、带方向；每张图只回答一个问题。
- **一致（Consistency）** — 所有团队、每一周长得一样，读者形成肌肉记忆，注意力留给「数字变了什么」而不是「这周排版怎么又变了」。

> 反向自检：如果一处设计是为了「好看 / 炫」，删掉它；如果是为了「让结论更快被读到」，保留它。

## 2. 页面信息架构

🔒 固定骨架，自上而下：

```
┌─ 抬头 eyebrow ──────────────  团队 · 周期 · 数据口径（小字、克制）
├─ 一句话结论 headline ───────  本周核心判断（含「但是」）+ 一行关键数字摘要
├─ 三支柱 pillars ────────────  赢点 / 最大风险 / 值得加注  ← 叙事锚点
├─ 分隔 "经营全景 · 明细" ────  从「结论」切到「证据」的视觉换挡
├─ KPI 计分卡 scorecard ──────  4–6 个核心指标全景
├─ 图表行 charts ─────────────  趋势折线 + 结构环图
├─ 表格行 tables ─────────────  分区/分段明细 + 风险阈值
├─ 下周三步走 next steps ─────  2–4 条行动（责任人 / 截止）
└─ 页脚 footer ───────────────  口径 / 免责 小字
```

**为什么这个顺序**：头部回答「这周怎么样？」（结论），中部回答「凭什么这么说？」（证据），尾部回答「那要做什么？」（行动）。读者可以在任意深度停下，且越往下越细。

可裁剪：业务线若没有「分区」就换成你的主切片（产品线/渠道/客户分层）；没有漏斗就不放。**但头部的「结论 + 三支柱」不要删**——它是这份报告区别于「一堆图表」的关键。

**模块可选与裁剪（本次约定）**：除头部「结论 + 三支柱」外，中下部各模块按业务需要取舍，常见配置：

- **下周三步走 / 行动**：可选模块。若本期无需单列行动项，可整块省略（不要为凑结构编造行动）。
- **风险与关注项表**：可选模块。风险若已在头部三支柱与业务线明细的「本周总结」中体现，可不再单列，避免重复。
- **业务线明细表（推荐）**：本次将「分区/分段表」升级为**业务线经营表现表**，按本周收入（或你的核心结果指标）降序排列，列建议含：业务线 / 周收入 + WoW / 交易量·TTV + WoW / **本周总结**。
  - 🔒 **「本周总结」列直接取自源周报的「本周解读 / 归因」原文**，不在大盘里重新归纳或新增结论；不同业务线交易量口径不同（TTV / Payout / 日均量等）须在单元格内标注口径，无统一口径的标 `—`。
- **流动性 / 成本明细表**：当源周报含对冲成本、Orderbook 流动性、PSP 渠道成本等内容时，以表格形式补在业务线明细表之后（成本类着色见 §4）。
- **OKR 达成矩阵**：当需要展示「距全季目标的进度」时，以横条 + 基准线（如时间进度）的矩阵列在末尾；季度级 QoQ 与周度 WoW 是两个口径，不要混为一谈。

🔒 **策展视图须声明**：当大盘为「CEO 摘要」而**有意省略**源周报的某些板块（如流动性 / PSP 明细）时，必须在页脚注明"本视图为策展摘要，XX 明细见完整周报"，不得静默丢板块让读者误以为是全量。一旦把被省略的板块补回正文，同步移除该免责。

## 3. 配色系统

🔒 **一个主色原则**：中性灰阶承载 95% 的界面，**只有一个主色**承载强调。再加一个语义负向色。多一个亮色，克制感就垮。

| 角色 | 变量 | 默认值 | 用途 |
|---|---|---|---|
| 背景 | `--bg` | `#f6f7f8` | 页面底 |
| 卡片 | `--panel` | `#ffffff` | 卡片 / 表格底 |
| 主文字 | `--ink` | `#16181d` | 标题、数值 |
| 次文字 | `--sub` | `#64748b` | 标签、说明、轴标签 |
| 分隔线 | `--line` | `#e7eaee` | 边框、表格线 |
| **主色** | `--accent` | `#0f766e`（深青） | 强调、正向、折线、环图主段、行动按钮 |
| 主色浅底 | `--accent-soft` | `#e6f1ef` | tag 背景 |
| 正向 | `--good` | `#0f766e` | 上涨且是好事 |
| 负向 | `--bad` | `#c2410c`（暖橙红） | 下跌 / 变坏 |

**为什么负向用暖橙红 `#c2410c` 而不是纯红 `#ff0000`**：纯红过于警报化，一页纸里出现几个就显得「到处着火」。暖橙红在白底对比度达标（≈4.9:1）、语义足够「注意」，又不破坏克制基调。

**换肤**：改 `--accent` 一个变量，全篇主色统一迁移。环图的同色阶（如 `#0f766e / #14b8a6 / #5eead4`）需相应换成你主色的明度梯度。**不要改 `--bad` 和状态 pill 的颜色**——它们的对比度是按 WCAG 调过的（见 §10）。

**状态 pill（信号灯）配色**（🔒 已按对比度锁定）：

| 状态 | 背景 | 文字 | 含义 |
|---|---|---|---|
| 绿 `.g` | `#dcf2ec` | `#0f766e` | 健康 / 达标 |
| 黄 `.a` | `#fcefd9` | `#92400e` | 关注 / 偏高 |
| 红 `.r` | `#fbe3d6` | `#9a3412` | 超标 / 告警 |

## 4. 涨跌与符号

这是最容易做错、且做错会显得不专业的一节。🔒

**两个独立通道，不要混淆**：

- **箭头 `▲ / ▼` 表示「数字本身的方向」**（变大 / 变小）。
- **颜色（绿/主色 vs 橙红）表示「这是好事还是坏事」**。

大多数指标二者一致：交易量 `▲ +12.3%`（绿）、用户流失 `▼`（绿，下降是好事）。

⚠️ **冲突场景**：当方向与好坏相反时，箭头会误导，**去掉箭头，只用颜色**。

- 经典反例：**获客成本上升**。成本变大是坏事（应红色），但它在「上升」——
  - 标 `▲` → 红色上箭头，读者第一眼会以为「绿色般的好上涨」，语义打架；
  - 标 `▼` → 更糟，把「成本升高的坏消息」标成「下降」，**方向错误**。
  - ✅ 正确做法：**红色文字、无箭头**，如 `<div class="delta" style="color:var(--bad)">破 $50 红线 · 净新增 -5.2%</div>`。让颜色传达「坏」，文字传达事实，不用箭头。

**实现**：箭头用 CSS 伪元素，只挂在确实「方向=好坏」的元素上：

```css
.up{color:var(--good)} .down{color:var(--bad)}
.up::before{content:"▲ "} .down::before{content:"▼ "}
```

需要「红色无箭头」时，**不要用 `.down` 类**（它会自动加 ▼），改用内联 `style="color:var(--bad)"` 或一个无 `::before` 的类。

**成本类指标的专用着色（本次约定）**：对冲成本、PSP 渠道成本、点差等「越低越好」的指标，方向与好坏天然相反，统一用一组**无箭头**的语义类，避免每次写内联样式：

```css
.cost-good{color:var(--good)}   /* 成本下降 = 好 */
.cost-bad{color:var(--bad)}     /* 成本上升 = 坏 */
.cost-flat{color:var(--sub)}    /* 持平 / 无明显异动 = 中性 */
```

判定口径：若源周报已把某成本变动判定为「无明显异动 / 无需跟进」，即使数值有升降也用 `.cost-flat`（中性），把红/主色留给真正被判定为异常的成本项——与 §1 的「颜色是稀缺告警信号」一致。

**其它符号约定**：

- 环比/同比的数值**始终带符号**：`+12.3%`、`-5.2%`，不要写裸 `12.3%`。
- 单位用小字弱化（`48.2` 大、`亿$` 小），让数值主导视觉。
- 「无数据 / 不适用」一律写 `—`（em dash），不要留空、不要编造。

## 5. 数字与排版

- **字体** 🔒：系统无衬线，中文优先 `PingFang SC / Microsoft YaHei`：
  `font-family:-apple-system,"PingFang SC","Microsoft YaHei",Segoe UI,sans-serif`。
- **所有数字用等宽数字** 🔒：给每个含数值的元素加 `font-variant-numeric:tabular-nums`。理由：周报是给人逐列扫的，等宽让 `48.2 / 1,240 / 1.84` 纵向对齐，参差的比例字会拖慢阅读。
- **数量级与单位**：大数用「亿/万」收敛位数（`48.2 亿$` 而非 `4,820,000,000`），单位贴在数值后并弱化字号/颜色。同一份报告内单位口径统一。
- **字号梯度**（参考，可整体缩放）：headline 30px / 支柱数字 34px / KPI 数字 23px / 卡片标题 14px / 正文 13–14px / 小字 11–12px。
- **强调**：加粗 700–800 只给数值和小标题；正文 400–600。斜体几乎不用。

## 6. 图表选择

🔒 按**数据形状**选图，不按「想要花哨」选图。决策表：

| 数据形状 | 选择 | 约束 |
|---|---|---|
| 一个指标随时间变化（≤12 期） | **折线 / 面积图** | 高亮最新点并标值；轴标签达标对比度 |
| 卡片内的微型趋势 | **火花柱 sparkline** | 8 根左右，末根用主色/语义色 |
| 部分占整体（构成） | **环图 donut** | **≤5 段**；超过 5 段改横向条形 |
| 多属性的分类明细 | **表格** | 数字右对齐、等宽、首列左对齐 |
| 转化 / 漏斗 | **横向漏斗条** | 标每级转化率 |
| 状态 / 健康度 | **信号灯 pill** | 颜色 + 文字双编码 |
| 单个大数 + 环比 | **KPI 卡** | 不必配图 |

**禁用** 🔒：多于 5 片的饼图、双 Y 轴、3D、渐变堆叠、为区分而堆砌的高饱和色、与数据无关的装饰性图形。理由都一样——它们增加解码成本、稀释结论。

**一图一问**：每张图只回答一个问题。趋势图回答「在涨还是跌」，环图回答「钱从哪来」。不要把两个问题塞进一张图。

🔒 **没有趋势数据就不画趋势**：折线/火花柱必须基于**真实的多期序列**。当某指标只有当期值 + 环比、没有历史序列时，**只展示当期数字，不要凭观感编造一条迷你趋势**（哪怕只是装饰性的火花柱）。这与「缺数据标 `—`、不编造」是同一条诚实底线。序列点数少（如仅 3 期）也照实画 3 个点，不要为了"好看"凑到 8 点。同一页里允许「部分 KPI 有火花线、部分只有数字」的不齐整——诚实优先于视觉对称。

🔒 **构成图须排除一次性/非经常项的误导**：环图等"构成/占比"图表反映的是**结构性**关系。当某一段（如某业务线收入）大部分来自一次性事件（一次性分润、单笔大客户、追溯结算）时，必须在该段图例或 `hint` 中标注「含一次性」，否则会让读者把一次性占比误读为常态结构。必要时并列给「剔除一次性后」的对照。此规则与头部 headline 对一次性增量的定性须一致。

## 7. 组件库

每个组件的完整 HTML/CSS 见 `templates/report-template.html`，下面是用法要点。

- **抬头 eyebrow** — 三栏小字（团队 / 周期 / 口径），下方一条 `2px solid var(--ink)` 实线收口。克制、不抢戏。
- **一句话结论 headline** — `<h1>`，30px/800。用 `<em>`（主色、非斜体）高亮正向词、`<span class="warn">`（负向色）高亮风险词。下接一段 `.sub` 小字罗列关键数字。
- **三支柱 pillar** — 三等分卡片，每个含：彩色 tag（赢点/风险/押注）、`<h2>` 标题、大号 hero 数字、一行 delta、一段说明。可选 8 根 mini 火花柱——🔒 **仅当该支柱指标确有多期真实序列时才画**，无序列则省略，不要凭观感编造（见 §6）。语义类 `.w/.r/.b` 决定火花末根颜色。
- **分区标签 seclabel** — `<h2>` + 一条延伸到行尾的细线，作为「结论 → 明细」的换挡。🔒 用真 `<h2>` 而非 `<div>`，否则 PDF 大纲/读屏器抓不到。
- **KPI 卡** — 标签 + 大数值（带小单位）+ 一行环比（`.up/.down`）。一行 4–6 个。
- **折线卡 / 环图卡** — 见 §8 几何。卡片标题用 `<h3>`，副标题 `.hint` 是**口径说明**（如「单位：亿美元」「按业务线拆分」），**不是再下一次结论**（见 §11）。
- **表格** — `<thead>` 用次文字色小字；数据行数字右对齐 + 等宽；状态列放 pill。
- **信号灯 pill** — `.pill.g/.a/.r`，颜色 + 文字（健康/偏高/超标）双编码。
- **行动步骤 step** — 时效徽章（即刻/本月，红=紧急）+ 行动描述（动词开头、加粗关键词）+ 右对齐的「责任人 · 截止日」。

## 8. 图表几何（SVG 公式）

数据一变，**必须重算坐标**，不能留旧值。纯内联 SVG，无 JS 依赖。

### 折线 / 面积图

画布 `viewBox="0 0 600 220"`，绘图区 x∈[20,540]、y∈[基线170, 顶部~20]。

- **N 个点的 x**：`x_i = 20 + i * (520 / (N-1))`，i=0..N-1。
- **y 映射**：给数据留点上下边距。设 `vmin/vmax` 为略低于最小、略高于最大的整数，
  `y_i = 170 - (v_i - vmin) / (vmax - vmin) * 150`。
- **面积**：折线 points 末尾接 `540,170 20,170` 闭合到基线，`fill:var(--accent);opacity:.07`。
- **最新点**：半径加大并在其上方 `text` 标值。
- **轴标签** 🔒：`fill="#64748b"`（≈4.76:1，达 AA），**不要用更浅的 `#94a3b8`**（仅 2.56:1，打印/缩放后读不清）。

### 环图 donut

半径 `r=70`，周长 `C = 2πr ≈ 439.8`，`stroke-width:22`，整体 `transform="rotate(-90 90 90)"` 让起点在 12 点方向。每段一个 `<circle>`：

- 段长 `dash_k = pct_k * C`；`stroke-dasharray="dash_k  (C - dash_k)"`。
- 偏移 `stroke-dashoffset = -(前面所有段长之和)`，第一段为 0。
- 🔒 自检：所有 `dash_k` 之和应 ≈ C（439.8），各百分比之和 = 100%。

**示例**（62/16/14/8）：
```
62% → dash 272.7,  offset 0
16% → dash 70.4,   offset -272.7
14% → dash 61.6,   offset -343.1   (272.7+70.4)
 8% → dash 35.2,   offset -404.7   (343.1+61.6)
```
图例颜色顺序 🔒 必须与各段 `stroke` 颜色一一对应。

### 火花柱 sparkline

8 个 `<span>` 用 `height:NN%` 表高度（flex 等宽），末根换主色/语义色。高度是相对值即可，不必精确映射。

## 9. 打印 / 导 PDF

🔒 周报会被打印、会被导成 PDF 转发，所以打印样式是规范的一部分，不是可选项。在 `</style>` 前放：

```css
@media print{
  *{ -webkit-print-color-adjust:exact; print-color-adjust:exact; }  /* 否则状态色底被丢，红/绿语义全失 */
  @page{ size:A4 landscape; margin:10mm; }
  body{ background:#fff; }
  .page{ max-width:none; padding:0; }
  .pillars,.pillar,.card,.kpi,.next,.next .step,table tr{ break-inside:avoid; page-break-inside:avoid; }  /* 防卡片被跨页拦腰截断 */
  .pill,.pillar .tag,.foot,.next .step .a,.next .step .own{ font-size:12px; }  /* 打印缩放下小字保底 */
}
```

四件事缺一不可：① 强制保留底色（状态 pill 靠底色区分红黄绿）；② 防跨页截断；③ 固定 A4 横向 + 页边距；④ 小字保底。

## 10. 可访问性

不是合规姿态——周报常被打印、被投屏、被中老年高管在小字下阅读，对比度和层级直接影响「能不能读」。🔒

- **文字对比度 ≥ 4.5:1**（WCAG AA 正文）。小号 pill（11px）不享受「大字 3:1」豁免，必须 4.5:1。已锁定的达标值见 §3 状态 pill 表（红 `#9a3412`/5.94:1、黄 `#92400e`/6.24:1、绿 `#0f766e`/4.68:1）。轴标签见 §8（`#64748b`）。
- **不靠颜色单独表意** 🔒：状态永远「颜色 + 文字」双编码（红底 + 「超标」二字），色盲读者也能读。
- **标题层级单调 `h1→h2→h3`** 🔒：不跳级（不要 h1 直接到 h3）。本规范的层级：`h1`=一句话结论 → `h2`×3=三支柱 → `h2`=「经营全景·明细」分区 → `h3`×N=各卡片标题 → `h2`=下周三步走。向上回跳（h3→h2 进入新区）允许；向下跳级不允许。理由：PDF 书签和读屏大纲按层级嵌套，跳级会乱。
- **语义标签**：分区标题用 `<h2>` 而非样式化 `<div>`；表格用 `<thead>`。

## 11. 反模式与克制清单

逐条都是「显得不专业 / 伤克制」的常见错误：

- ❌ **头部就是一串数字**。headline 要给判断（含「但是」），不是 KPI 的复读。
- ❌ **同一个数字在相邻位置反复出现**。「结论 → 明细」复述一次可以（如交易量出现在支柱和 KPI 卡），但图表副标题不要再把头部已下的结论喊一遍——副标题写**口径**（「按业务线拆分 ｜ 单位：万美元」），不写「理财升至 16%，第二曲线成型」。
- ❌ **口径前后打架**。当前值与阈值要分清：「CAC 升至 $58、突破 $50 红线」对；「CAC 突破 $58 红线」错（把当前值当成了阈值）。头部、支柱、KPI、表格、图表必须讲同一组数字、同一套阈值。
- ❌ **第二个亮色**。多一个高饱和色，整页就「花」。需要区分时用主色的明度阶或中性灰。
- ❌ **箭头与语义打架**（见 §4 成本上升案例）。
- ❌ **满页红绿、稀释告警**。把所有同向波动自动着色，让真正需要跟进的那一项淹没在红海里。常规回调用中性灰，红色只留给头部判定为「需跟进」的项（见 §1 克制原则）。
- ❌ **编造趋势 / 假火花线**。没有多期序列就只给当期数字，不要为视觉对称凑一条迷你趋势或补到 8 点（见 §6）。
- ❌ **一次性收入混入结构图**。构成/占比图里把一次性事件当成常态结构展示而不标注，误导读者（见 §6）。
- ❌ **静默删板块**。策展摘要省略了源周报的板块却不在页脚声明，读者误以为是全量（见 §2）。
- ❌ **在大盘里重新归纳业务线解读**。明细表「本周总结」应照搬源周报原文，不另起结论（见 §2、§7）。
- ❌ **图表配文再下一次结论**、**为装饰而画图**、**多片饼图/双轴/3D**。
- ❌ **裸数字无符号无单位无对齐**。
