# Markdown 源 — 用 Markdown 周报作为数据源

本 skill 的**主数据源是一份 Markdown 周报文档**。团队成员（或上游系统、飞书文档导出）写一份正常的 markdown 周报，skill 读它 → 映射到[数据契约](data-schema.md)的字段 → 按 [design-spec](design-spec.md) 渲染成一页纸 HTML。

> 🔑 **分工**：Markdown 只负责**内容与数值**（写了什么、多少、解读原文）。**所有呈现判断仍由 design-spec 在渲染时套用**——着色（CAC 自动 `cost-bad`、比率自动 `pt`、币价自动 `muted` + 颜色图例）、图表选择、排序（Others/一次性置底）、格式（小数位一致）、克制与策展。作者**不需要**在 markdown 里写颜色/类名/图表类型。

## 两种 Markdown 源

- **(A) 结构化「数据 markdown」**：按下面的推荐结构写，字段映射确定、最省事。`templates/source.example.md` 就是一份可直接渲染的样例。
- **(B) 自由式「团队已有周报」**：一份没按本结构写的 markdown 周报。这时按**语义抽取**：找到结论、关键指标、表格/序列，映射到字段；缺的标 `—` 不编造；抽取不出或口径不清时**向用户澄清**，不要猜。大盘是源周报的**策展摘要**——主动省略的板块按 design-spec §2 在页脚声明。

## 映射表（Markdown → 数据契约字段）

| Markdown 元素 | → 数据契约（data-schema.md） | 备注 |
|---|---|---|
| YAML front-matter `team/subtitle/period/caveat/accent` | `identity.*` | `accent` 可省，默认深青 |
| `## 结论` 段落 | `headline.verdict`（+ 自动生成 `summary` 或取 KPI 行） | 含「但是」；关键词渲染时上语义色 |
| `## 赢点/风险/押注` 三条 | `pillars[]`（kind=win/risk/bet） | 一行五段：标题｜值+单位｜delta｜趋势(逗号8值或「无」)｜说明。**趋势=无 → 不画火花线**（design-spec §6） |
| `## KPI` 表 | `kpis[]` | 列：指标/值/单位/环比。方向由渲染时按 §4 判定树定（成本类 cost-bad、比率 pt、价格 muted…） |
| `## 趋势` 的 `标签:`/`数值:` 两行 | `trend.labels/values`（+ title/hint） | 数值据此重算 SVG 坐标。**< 3 个数值 → 不画折线**，只给 KPI+环比 |
| `## 收入构成` 列表 + `单位:`/`中心:` | `composition.segments/centerValue` | **≤5 段**，超过改条形；某段含一次性须在文案标注 |
| `## 分区域…` 表（可改名为你的切片） | `breakdown.columns/rows` | 「本周总结」列**照搬源周报原文**，不重新归纳（§2/§7）。Others/一次性行渲染时置底 |
| `## 风险与关注项` 表 | `risks.rows` | 当前值与阈值分清 |
| `## 下周三步走` 列表 | `nextSteps[]` | 一行三段：时效｜行动｜责任人·截止。时效含「即刻/紧急」→ 红徽章。整块可省略 |
| `## 页脚` | `footer` | 口径/免责 |

未出现的小节 = 该模块缺省（按 design-spec 可裁剪；缺数据标 `—`，不编造）。

## 解析与诚实规则 🔒

1. **只搬不造**：markdown 里没有的数值/序列，标 `—`，绝不编造。趋势序列必须来自源（`数值:` 行或表格列），没有就不画趋势（§6）。
2. **本周总结照搬原文**：业务线表的「本周总结」列、以及自由式源里的解读，直接引用源 markdown 的归因原文，不在大盘里另下新结论（§2/§7）。
3. **渲染时套规则，不靠源指定**：着色/图表/排序/格式全部按 design-spec 判定——
   - 比率指标（成功率/拒付率/费率…）环比一律 `pt`（§5）；
   - 成本/越低越好指标用 `.cost-good/.cost-bad`（§4）；
   - 币价/代币涨跌字段用 `.muted` + 页脚 `.color-legend`（§3）；
   - 构成/明细里 Others 与一次性置底（§6）；同一指标小数位整页一致（§5）。
4. **策展即声明**：大盘是源 markdown 的克制摘要。被有意省略的源板块，按 §2 在页脚注明「本视图为策展摘要，XX 明细见完整周报」。
5. **歧义澄清**：字段对不上、口径不清（如交易量是真实额还是名义额）、基线不明（环比/同比）时，**问用户**，不要替源做假设。

## 工作流程（markdown 源 → HTML）

1. **读取** markdown 源（用户给文件路径或粘贴正文）。判断是结构化(A)还是自由式(B)。
2. **映射**到数据契约字段（按上表 / 语义抽取），缺则标 `—`。
3. **渲染**：复制 `templates/report-template.html`，按 design-spec 套用全部呈现规则，重算 SVG 几何。
4. **自检**：过 SKILL.md 的 Quality checklist；策展省略要在页脚声明。
5. **交付**：给出 HTML 路径 + 查看/导 PDF 方式；如有澄清项一并列出。
